Modelagem de workflows científicos

Terminologia básica
- modelo: define atividades a serem realizadas no workflow para que um
  objetivo científico seja alcançado.
- atividade: é uma unidade atômica de trabalho em um modelo de workflow.
- conector: é usado para definir alguma relação de precedência ou algum
  tipo de transferência de dados entre atividades.
- instância : uma execução particular

Modelo de workflow
- esteabelece a ordem de atividades
- pode usar um arcabouço formal ou linguagem de especificação de
  workflow.

Atividade
- unidade atômica de trabalho
- pode ser:
    - manual
    - automática
    - semiautomática

Conector
- define alguma relação de precedência ou algum tipo de transferência de
  dados entre atividades

Instcia de workflow
- execuão particula de um modelo de workflow

Workflow de negócio x científicos
- Workflow de negócio
    - automatizam processo industriais e gerenciais
    - desenvolvimento apoiado por comunidades e coalizões, envolvendo
      indústria e academia
- Workflow científico
    - similar ao de negócios
    - a comunidade científica não adotou os padrões desenvolvidos para
      os worflows de negócios.


Padronização e interoperabilidade
- W. de negócio
    - processos inter-organizacionais
    - demandam interoperabilidade de ferramentas e padronização
- W. científicos
    - não existem padrões
    - pouca interoperabilidade

Perspectivas para a modelagem de workflows científicos
- fluxo de controle: descreve a ordem de execução
- fluxo de dados: descreve as tranferências de dados entre as atividades
